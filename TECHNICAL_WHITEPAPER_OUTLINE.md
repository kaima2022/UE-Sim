## 🏗️ 3. 系统架构设计

### 3.1 整体架构概览
```
应用层
    ↓
SES (Semantic Sub-layer)     ← 语义管理层
    ↓
PDS (Packet Delivery)       ← 包分发层
    ↓
PDC (Delivery Context)      ← 传输上下文层
    ↓
ns-3网络层                  ← 仿真框架层
```

### 3.2 SES (Semantic Sub-layer) 设计
#### 3.2.1 核心功能
- 消息类型处理和路由决策
- 语义头格式管理和解析
- 端点寻址和授权机制
- MSN (Message Sequence Number) 表管理

#### 3.2.2 关键组件
- **SesManager**: 语义管理器核心实现
- **OperationMetadata**: 操作元数据管理
- **MsnEntry**: MSN表条目定义

#### 3.2.3 性能优化
- 微秒级操作响应机制
- 高效的内存池管理
- 零拷贝数据路径优化

### 3.3 PDS (Packet Delivery Sub-layer) 设计
#### 3.3.1 核心功能
- 包分发和路由决策
- PDC实例管理和协调
- 性能统计收集和分析
- 流控制和拥塞控制

#### 3.3.2 关键组件
- **PdsManager**: 包分发管理器
- **PdsHeader**: 协议头部处理
- **PdsStatistics**: 性能统计模块

#### 3.3.3 流控制机制
- 基于信用的流控制算法
- 动态队列管理策略
- 拥塞检测和响应机制

### 3.4 PDC (Packet Delivery Context) 设计
#### 3.4.1 双层架构
- **IPDC (Unreliable PDC)**: 最佳努力交付
- **TPDC (Reliable PDC)**: 可靠交付保证

#### 3.4.2 IPDC实现
- 轻量级包传输机制
- 流控制和队列管理
- 最佳努力交付策略

#### 3.4.3 TPDC实现
- ACK/NACK确认机制
- RTO (Retransmission Timeout) 定时器
- 指数退避重传算法
- 可靠性保证机制

#### 3.4.4 RTO定时器设计
- 高精度定时器实现
- 自适应超时调整算法
- 并发定时器管理

### 3.5 网络设备集成层
#### 3.5.1 SoftUeNetDevice设计
- 完全符合ns-3 NetDevice接口
- 协议栈初始化和配置管理
- 数据包收发和处理逻辑

#### 3.5.2 SoftUeChannel实现
- 物理层传播延迟建模
- 带宽约束和干扰仿真
- 广播和多播传输支持

## 💻 4. 实现细节

### 4.1 开发环境和工具链
- **框架**: ns-3.44离散事件仿真器
- **语言**: C++17标准
- **构建系统**: CMake + Ninja
- **测试框架**: ns-3内置测试框架

### 4.2 模块化设计原则
#### 4.2.1 SOLID原则应用
- **单一职责**: 每个组件专注单一功能
- **开闭原则**: 通过接口实现扩展性
- **里氏替换**: PDC接口的继承体系
- **接口隔离**: 专一的接口设计
- **依赖倒置**: 依赖抽象而非具体实现

#### 4.2.2 代码组织结构
```
src/soft-ue/
├── helper/                 # Helper层
│   └── soft-ue-helper.h/cc
├── model/                  # 核心模型层
│   ├── ses/               # SES层实现
│   ├── pds/               # PDS层实现
│   ├── pdc/               # PDC层实现
│   ├── network/           # 网络设备层
│   └── common/            # 通用组件
├── test/                  # 单元测试
└── examples/              # 示例代码
```

### 4.3 关键技术实现

#### 4.3.1 内存管理优化
- **对象池模式**: 预分配内存减少动态分配
- **智能指针**: RAII保证内存安全
- **零拷贝优化**: 减少数据拷贝开销

#### 4.3.2 并发控制机制
- **无锁数据结构**: 减少锁竞争
- **原子操作**: 保证线程安全
- **事件驱动**: 基于ns-3事件调度

#### 4.3.3 性能监控和统计
- **实时统计**: 性能指标实时收集
- **可配置追踪**: 灵活的调试和监控
- **基准测试**: 标准化性能评估

### 4.4 代码质量保证
- **编码规范**: 严格的ns-3编码标准
- **文档覆盖**: 100%公共接口文档化
- **单元测试**: 完整的测试覆盖体系
- **持续集成**: 自动化构建和测试
