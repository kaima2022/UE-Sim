# Soft-UE ns-3 技术规格文档

## 项目概述

Soft-UE ns-3项目成功实现了Ultra Ethernet协议栈在ns-3离散事件仿真网络环境中的完整移植。项目实现了完整的代码编译、库生成和功能运行，提供了可靠的技术实现和功能支持。

## 架构设计

### 三层协议架构

项目采用Ultra Ethernet三层架构设计，每层承担明确的协议功能：

#### SES（Semantic Sub-layer）语义子层
SES层作为协议栈的最高层，负责处理语义层面的操作，包括操作类型识别和处理、消息序列号管理、操作元数据管理和扩展、端点寻址和授权验证以及错误响应生成和处理。

**核心组件**：
- **SesManager类**：负责操作队列、MSN表和统计功能的集成
- **ExtendedOperationMetadata类**：扩展基本的操作元数据，添加了ns-3仿真环境特有的节点ID、端点ID和操作ID字段
- **OperationMetadataFactory类**：提供工厂方法创建不同类型的操作元数据

#### PDS（Packet Delivery Sub-layer）包分发子层
PDS层作为中间层，负责数据包的分类、路由和分发管理。主要功能包括包类型识别和分类处理、PDC管理和分配、包头格式处理和生成、性能统计收集和分析以及拥塞控制和流量管理。

**核心组件**：
- **PdsManager类**：处理多种包类型
- **PdsStatistics类**：提供详细的性能统计功能，支持实时监控和历史数据分析
- **PdcAllocator类**：负责PDC资源的分配和管理

#### PDC（Packet Delivery Context）包分发上下文层
PDC层作为最底层，负责处理具体的传输上下文管理和数据传输。提供两种传输模式：

**IPDC不可靠传输**：
- 实现无确认传输机制，优先保证低延迟
- 提供资源快速分配和回收
- 适用于对实时性要求高的应用场景

**TPDC可靠传输**：
- 实现基于RTO的超时重传机制
- 序列号管理和乱序处理功能
- 提供可靠的数据传输保障

### ns-3框架集成设计

#### 网络设备层
**SoftUeNetDevice类**完全遵循ns-3的网络设备设计模式：
- 继承NetDevice基类，实现标准接口
- 支持ns-3属性系统和配置机制
- 集成Send、Receive、SetNode等核心方法
- 提供完整的MAC地址和MTU管理

#### 通道层
**SoftUeChannel类**实现了仿真网络的物理传输特性：
- 支持点对点和多点连接
- 可配置的传输延迟、带宽和丢包率
- 集成错误注入和性能监控功能
- 支持动态网络拓扑变化

#### Helper层
**SoftUeHelper类**为用户提供了友好的配置接口：
- 简化设备安装和网络部署流程
- 提供默认参数配置和自定义选项
- 支持批量操作和自动化部署

## 核心组件技术实现

### 操作元数据管理

#### OperationMetadata基础结构
OperationMetadata定义了协议操作的基本属性：
- **操作类型枚举**：SEND/READ/WRITE/DEFERRABLE四种类型
- **内存区域信息**：rkey、idempotent_safe标志
- **数据载荷信息**：地址、长度、立即数据
- **端点信息**：源/目标进程ID、作业ID、资源索引
- **操作标志**：相对寻址、优化头、立即数据标志

#### ExtendedOperationMetadata扩展实现
ExtendedOperationMetadata扩展了ns-3仿真所需的字段：
- **节点ID映射**：source/destination node ID
- **端点ID映射**：source/destination endpoint ID
- **操作时间戳**：创建时间跟踪
- **唯一标识符**：操作ID管理
- **数据包计算**：分片判断功能
- **元数据验证**：完整性检查

### 消息序列号管理

#### MSN表结构
MSNEntry类维护消息传输状态：
- **最后接收的包序列号**：乱序重组基础
- **消息期望总长度**：完整性检查
- **关联的PDC标识符**：上下文关联
- **支持乱序重组**：协议功能保证

#### MSN管理策略
实现高效的序列号管理机制：
- **动态序列号分配**：避免冲突
- **超时清理机制**：资源回收
- **内存使用优化**：高效存储
- **并发访问安全**：线程安全

### 协议头处理

#### PDS头格式
PdsHeader类实现完整的协议头处理：
- **多种头类型支持**：请求、响应、控制包等
- **可变长度头字段**：灵活处理
- **校验和计算验证**：数据完整性
- **序列化反序列化**：传输转换

#### 头类型系统
支持丰富的头类型枚举：
- **TSS**：UET加密头
- **RUD_REQ/ROD_REQ**：可靠传输请求
- **UUD_REQ**：不可靠传输请求
- **ACK/ACK_CC**：确认和拥塞控制确认
- **NACK**：否定确认
- **CP**：控制包

### 传输上下文管理

#### PDC分配算法
实现高效的PDC分配策略：
- **动态PDC池管理**：支持512个并发PDC
- **基于负载的智能分配**：优化资源利用
- **优先级队列调度**：QoS支持
- **资源回收防护**：防止泄漏

#### 并发控制机制
TPDC提供可靠的并发传输控制：
- **基于滑动窗口的流量控制**：防止拥塞
- **动态RTO计算调整**：自适应超时
- **快速重传和恢复**：提高效率
- **选择性确认支持**：减少重传

## 配置参数和性能特性

### 核心配置常量

#### 协议参数
核心协议参数：
- **MAX_MTU**：4096字节最大传输单元
- **MAX_QUEUE_SIZE**：512最大队列大小
- **MAX_PDC**：512最大PDC数量
- **Base_RTO**：100毫秒基础重传超时

#### UET协议配置
UET协议配置：
- **UET_Over_UDP**：1（UET运行于UDP之上）
- **UDP_Dest_Port**：2887（UET协议UDP端口）
- **Default_MPR**：8（默认最大包率）

#### 重传参数
重传参数：
- **RTO_Init_Time**：4（初始RTO倍数）
- **Max_RTO_Retx_Cnt**：5（最大重传次数）
- **NACK_Retx_Time**：4（NACK重传延迟）
- **Max_NACK_Retx_Cnt**：5（最大NACK重传次数）

## 错误处理和诊断

### 错误分类系统

#### 响应返回码
完整的错误响应体系：
- **RC_OK**：正常操作
- **RC_PARTIAL_WRITE**：部分数据写入
- **RC_NO_MATCH**：标签发送无匹配缓冲区
- **RC_INVALID_OP**：无效操作代码
- **RC_NO_BUFFER**：无可用的接收缓冲区

#### 安全相关错误
安全错误处理：
- **RC_INVALID_KEY**：RMA内存密钥无效/过期
- **RC_ACCESS_DENIED**：作业ID无访问权限
- **RC_SECURITY_DOMAIN_MISMATCH**：安全域验证失败
- **RC_INTEGRITY_CHECK_FAIL**：包完整性检查失败
- **RC_REPLAY_DETECTED**：检测到重放攻击

#### 系统错误
系统错误处理：
- **RC_INTERNAL_ERROR**：SES内部状态异常
- **RC_RESOURCE_EXHAUST**：系统资源耗尽
- **RC_PROTOCOL_ERROR**：协议违规
- **RC_ADDR_UNREACHABLE**：目标地址不可达
