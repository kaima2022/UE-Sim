# 实验运行指南

## 快速开始

### 1. 编译项目

```bash
cd /home/makai/Soft-UE-ns3

# 配置 (首次)
./ns3 configure --enable-examples --enable-tests

# 编译
./ns3 build
```

编译成功后，会生成: `build/lib/libns3.44-soft-ue-debug.so`

### 2. 运行主测试脚本

```bash
# 运行 200Gbps 压力测试
./ns3 run Soft-UE

# 或者带详细日志
NS_LOG="SoftUeStressTest=info" ./ns3 run Soft-UE
```

### 3. 运行 UEC 概念实验

```bash
# 运行 UEC 端到端概念实验（对应 07-UEC端到端概念实验与图解.md）
./ns3 run uec-e2e-concepts
```

若出现 `Couldn't find the specified program: uec-e2e-concepts`，说明 ns3 的节目录未包含该可执行文件，需先重新配置再编译：

```bash
./ns3 configure --enable-examples --enable-tests
./ns3 build
./ns3 run uec-e2e-concepts
```

或直接运行生成的可执行文件（路径以实际 build 目录为准）：

```bash
./build/scratch/Soft-UE-E2E-Concepts/ns3.44-uec-e2e-concepts-debug
```

### 4. 运行单元测试

```bash
# 运行所有 soft-ue 相关测试
./ns3 test soft-ue

# 运行特定测试
./ns3 test ses-manager-test
./ns3 test pds-manager-test
```

## 测试脚本位置

| 脚本 | 位置 | 用途 |
|------|------|------|
| 主压力测试 | `scratch/Soft-UE/Soft-UE.cc` | 200Gbps 端到端压力测试 |
| UEC 概念实验 | `scratch/Soft-UE-E2E-Concepts/uec-e2e-concepts.cc` | UEC 关键概念端到端演示 |
| SES测试 | `src/soft-ue/test/ses-manager-test.cc` | SES层单元测试 |
| PDS测试 | `src/soft-ue/test/pds-manager-test.cc` | PDS层单元测试 |
| 完整PDS测试 | `src/soft-ue/test/pds-full-test.cc` | PDS完整功能测试 |

## 主测试脚本参数

`scratch/Soft-UE/Soft-UE.cc` 的关键配置:

```cpp
struct StressTestConfig {
    uint32_t nodeCount = 2;              // 节点数量
    uint32_t packetSize = 9000;          // 包大小 (Jumbo帧)
    uint32_t packetCount = 100000;       // 包数量
    Time sendInterval = NanoSeconds(1);  // 发送间隔
    uint32_t maxPdcCount = 4096;         // 最大PDC数
    DataRate channelDataRate = DataRate("200Gbps"); // 通道速率
};
```

## 日志级别控制

```bash
# 查看 SES 层详细日志
NS_LOG="SesManager=debug" ./ns3 run Soft-UE

# 查看 PDS 层详细日志
NS_LOG="PdsManager=debug" ./ns3 run Soft-UE

# 查看所有 soft-ue 日志
NS_LOG="SoftUe*=info" ./ns3 run Soft-UE

# 组合多个日志
NS_LOG="SesManager=info:PdsManager=info:SoftUeNetDevice=debug" ./ns3 run Soft-UE
```

## 常见问题

### Q: 编译失败 "soft-ue module not found"
```bash
# 确保 CMakeLists.txt 正确配置
cat src/soft-ue/CMakeLists.txt

# 重新配置
./ns3 clean
./ns3 configure --enable-examples --enable-tests
./ns3 build
```

### Q: 运行时 "undefined symbol"
```bash
# 检查库链接
ldd build/lib/libns3.44-soft-ue-debug.so

# 设置库路径
export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:$(pwd)/build/lib
```

### Q: 接收包数为0
这是已知问题 (见T002测试结果)，接收回调机制需要修复。发送功能正常工作。
